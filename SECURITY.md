# Security Policy

## Supported versions

This repo has no application code — just docs and git submodule pointers.
There is nothing here to patch directly; `main` is the only tracked state.

## Reporting a vulnerability

If you find a vulnerability in one of the platform's services
([`schlussel`](https://github.com/zudaR107/schlussel),
[`schloss`](https://github.com/zudaR107/schloss),
[`kuvert`](https://github.com/zudaR107/kuvert),
[`tor`](https://github.com/zudaR107/tor)), please report it privately
through that repo's own Security tab, not here.

Report something here only if it's specific to this repo: for example, a
submodule pinned to a commit with a known, publicly-disclosed
vulnerability. Use GitHub's private reporting flow:

1. Go to the [Security tab](../../security) of this repository.
2. Click "Report a vulnerability".
3. Describe the issue.

This is a small, mostly-solo project, so response time is best-effort, not
contractual — but you can expect an initial reply within a few days.
